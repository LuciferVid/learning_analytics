% ---------- ABSTRACT ----------
\begin{abstract}
This document presents an end-to-end learning analytics research project implemented using machine learning and agentic AI workflows. The system first classifies student performance into At-risk, Average, and High-performing categories using Logistic Regression, then extends into an AI study coach that diagnoses learning gaps, retrieves educational resources, and generates personalized multi-week study plans. The final implementation is deployed through a Streamlit interface and supports structured recommendations with Markdown/PDF report export.
\end{abstract}

\begin{IEEEkeywords}
Data mining, machine learning, learning analytics, classification, feature engineering, agentic AI.
\end{IEEEkeywords}

% ---------- INTRODUCTION ----------
\section{Introduction}
Educational institutions increasingly rely on data-driven systems to identify student learning gaps and provide timely support. Early detection of underperformance can reduce long-term academic risk and improve intervention outcomes. This project addresses that need by combining a supervised classification model with an agentic recommendation layer that converts performance insights into actionable study plans. The goal is to build a publicly deployable system that supports both analytics and personalized coaching.

% ---------- RELATED WORK ----------
\section{Related Work}
Prior Educational Data Mining (EDM) studies have applied classification methods such as Logistic Regression, Decision Trees, SVMs, and ensemble learners to predict academic outcomes from demographic and exam features. More recent work includes AI assistants that generate personalized learning recommendations. Our project follows this evolution by combining interpretable ML-based risk prediction with an agentic study coach pipeline.

% ---------- METHODOLOGY ----------
\section{Methodology}

\subsection{Dataset Description}
The primary dataset contains 1,000 student records with attributes including gender, race/ethnicity, parental level of education, lunch type, test preparation course, and subject scores (math, reading, writing). A derived target variable (\textit{performance category}) is created from average score thresholds.

\subsection{Preprocessing}
Categorical variables are encoded using Label Encoding. Total score and percentage are computed from the three subject scores. Features used for model inference are standardized with StandardScaler. Missing values are validated during CSV ingestion in the dashboard workflow.

\subsection{Feature Engineering}
We derive:
\begin{itemize}
    \item \textbf{Total score}: sum of math, reading, writing.
    \item \textbf{Percentage}: mean of the three subject scores.
    \item \textbf{Performance category}: At-risk ($<$50), Average (50--79), High-performing ($\geq$80).
\end{itemize}
These engineered variables support both supervised classification and downstream recommendation logic.

\subsection{Models Used}
The system includes two modeling components:
\begin{itemize}
    \item \textbf{Logistic Regression}: baseline supervised model for student risk/performance classification.
    \item \textbf{Agentic LLM layer}: free/open-source generation workflow that transforms diagnosis results into structured study plans with fallback behavior if model inference is unavailable.
\end{itemize}

\subsection{Training and Validation}
The dataset is split 80/20 for train-test evaluation. Performance is measured using Accuracy, Precision, Recall, and F1-score. We prioritize clear baseline interpretability for educational use while supplementing outcomes with rule-based and agent-generated recommendations.

\subsection{Milestone 2 Functional Results}
The deployed AI study coach generates the required structured output for user-defined goals:
\begin{itemize}
    \item Learning diagnosis (risk, strengths, and gaps),
    \item Personalized study plan,
    \item Weekly/milestone goals,
    \item Recommended resources with URLs,
    \item Progress feedback and next steps.
\end{itemize}
The system also supports session-level memory and downloadable Markdown/PDF coaching reports.

\subsection{Qualitative Observations}
The agentic layer improved practical utility by converting prediction outputs into actionable weekly plans. The fallback planner ensured stable behavior when LLM calls were unavailable, preserving output structure and usability.

% ---------- DISCUSSION ----------
\section{Discussion}
The baseline classifier provides effective risk stratification but remains limited by feature granularity and class imbalance. Integrating an agentic recommendation layer addresses this gap by adding goal-aware planning and retrieval-based resource support. However, recommendation quality may vary based on LLM availability, latency, and retrieved resource relevance. Future improvements can include richer learner behavior signals, adaptive difficulty, and persistent long-term memory.

% ---------- CONCLUSION ----------
\section{Conclusion}
This project demonstrates a full transition from ML-based learning analytics to an agentic AI study coach in a single deployable system. Logistic Regression provides interpretable baseline predictions, while the agentic layer generates structured, personalized study plans aligned with student goals. The final application meets milestone requirements with free/open-source compatible tooling and public deployment readiness.
